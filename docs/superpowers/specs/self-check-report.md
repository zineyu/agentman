| RISK-1 | Webhook需要Daemon暴露HTTP服务，但Daemon可能在内网/NAT后 | 中 | Phase 1使用轮询模式，Webhook作为可选增强 |

### 6. 飞书Base限制检查

| 限制项 | 影响 | 应对 |
|--------|------|------|
| API限流（约20 QPS） | 中 | 轮询间隔30s足够，批量操作 |
| Webhook延迟（1-5s） | 低 | 轮询模式下不依赖Webhook |
| 工作流执行次数限制 | 中 | 避免高频触发，关键逻辑放Daemon |
| 单表记录数上限 | 低 | 任务量不太可能达到上限 |
| 附件大小限制 | 低 | 执行日志存文本，大日志可分段 |
