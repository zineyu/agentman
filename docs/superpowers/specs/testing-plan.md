# Agentman 测试方案

## 测试分层

### 1. 单元测试（Unit Tests）
目标：验证单个模块的纯逻辑，无外部依赖

| 模块 | 测试内容 | 测试方式 |
|------|---------|---------|
| `config.rs` | 配置解析、验证、默认值 | 纯Rust测试 |
| `models/` | 序列化/反序列化、枚举转换 | 纯Rust测试 |

| `agent/` | AgentFactory创建、适配器类型匹配 | Mock适配器 |
| `utils.rs` | 工具函数 | 纯Rust测试 |

### 2. 集成测试（Integration Tests）
目标：验证模块间协作，使用 mock HTTP server

| 场景 | 测试内容 | 工具 |
|------|---------|------|
| BaseClient | Token获取、缓存、重试 | mockito |
| BaseClient | 任务查询、状态更新 | mockito |
| TaskExecutor | 完整执行流程（mock agent） | mockito + 临时目录 |

### 3. 端到端测试（E2E Tests）
目标：在真实飞书Base上验证完整流程

| 场景 | 操作步骤 | 预期结果 |
|------|---------|---------|
| 正常任务流 | 创建Agent任务 → Daemon执行 | 状态：待办→进行中→待审核 |
| 审核通过 | 审核人通过 | 状态变为"已完成" |
| 审核驳回 | 填写驳回理由 | 工作流重置为"待办"，Daemon重试 |
| 重试上限 | 连续驳回3次 | 状态变为"已取消" |
| 催办过滤 | Agent任务截止前1天 | 不发送催办 |
| 人类催办 | 人类任务截止前1天 | 发送催办通知 |

---

## 测试环境准备

### 单元/集成测试
```bash
cd agentman-daemon
cargo test
```

### 端到端测试
1. 准备测试用的 Base token
2. 创建测试任务（手动或通过脚本）
3. 运行 Daemon（`cargo run -- --config test-config.toml`）
4. 观察任务状态流转

## 关键测试用例

### TC-001: 配置默认值
验证默认配置生成正确，且缺失必填项时报错。

### TC-002: 任务模型序列化
验证 Task 结构体与 Base API JSON 的双向转换。



### TC-004: Agent适配器选择
验证 AgentType 到适配器的正确映射。

### TC-005: BaseClient Token缓存
验证 token 缓存命中、过期刷新、并发安全。

### TC-006: 任务执行成功流
Mock Base API，验证：
1. 获取待办任务
2. Agent执行成功
4. 状态更新为"待审核"
5. 执行日志写入

### TC-007: 任务执行失败重试
Mock Agent执行失败，验证：
1. 失败状态回写
2. 重试计数增加
3. 达到上限后变为"已取消"

### TC-008: 审核驳回重试
模拟审核驳回场景，验证：
1. 驳回理由追加到描述
2. 清空驳回理由字段
3. 状态重置为"待办"
4. 重新执行（不重复准备工作目录）

### TC-009: 工作流-审核驳回
在真实Base上测试：
1. 创建任务并执行到"待审核"
2. 审核人填写驳回理由
3. 验证工作流自动重置状态
4. Daemon重新执行

### TC-010: 工作流-催办过滤
在真实Base上测试：
1. 创建Agent任务，设置截止时间为明天
2. 验证不收到催办通知
3. 创建人类任务，设置截止时间为明天
4. 验证收到催办通知
